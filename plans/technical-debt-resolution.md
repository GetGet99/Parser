### 7. Performance: O(n²) String Building in RegexRunner

**Status:** ✅ **Completed** — replaced `matchedText += c` with `StringBuilder` in both `Next` and `NextWithPosition` methods. Added `using System.Text;`. Also had to add `FullType.cs` to both source generator projects (moved from deleted SyntaxCreator).

**Problem:** `Get.RegexMachine.Shared/RegexRunner.cs` line 16:
```csharp
matchedText += c
```
Each match concatenation creates a new string, O(n²) for long inputs.

**Resolution:** Replace with `System.Text.StringBuilder` or reuse an `ArrayPool<char>` / `Span<char>` buffer.

**Effort:** Small (30 min). Localized change. Should add perf regression test.

### 9. Unfinished EOF Handling in Lexer

**Status:** ✅ **Completed** — 2026-06-06

**Problem:** `Get.Lexer/LexerBase.cs` line 168:
```csharp
yield break; // TODO: Handle EOF
```
When the lexer reaches EOF during tokenization, it simply stops instead of producing a proper EOF token.

**Resolution:**
Added `protected virtual IToken<TTokenEnum>? GetEOFToken()` — default returns `null` (backward compat). When the input is exhausted, `GetTokens()` calls this and yields the result if non-null. Derived classes override to return a terminal EOF token with the current position.

**Effort:** Small (30 min).

### 10. Mixed Target Frameworks

**Problem:** `Get.Lexer` targets `net8.0` while `Get.Parser` and `Get.RegexMachine` target `netstandard2.0`. This prevents using the lexer in .NET Framework consumers and forces PolySharp polyfills.

**Resolution:** Decide on a single target:
- Option A: Move everything to `netstandard2.0` (broadest compatibility).
- Option B: Multi-target (`netstandard2.0;net8.0`).
- Option C: Move everything to `net8.0` (drop legacy support).

**Effort:** Medium (2-4 hours). Mostly csproj changes + testing. Option B is safest.

### 12. Thread Safety: ThreadLocal<RegexParser>

**Status:** ✅ **Completed** — 2026-06-06

**Problem:** `Get.RegexMachine.Shared/RegexParser.cs` line 14:
```csharp
static readonly ThreadLocal<RegexParser> Instance = ...
```
The `CreateEmptyNFAState` delegate property is mutable. If mutated after the thread-local is initialized, changes may not be visible to other threads (or worse, may race).

**Resolution:**
Made `CreateEmptyNFAState` a property with private setter. Added `Parse(string, Func<INFAState>)` overload that accepts the factory as a parameter, eliminating the mutation race window. Updated `RegexCompiler.NFA.cs` caller.

**Effort:** Small (30 min).
