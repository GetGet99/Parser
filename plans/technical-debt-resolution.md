### 8. `goto` in LR Parser Runtime

**Status:** тЬЕ **Completed** тАФ 2026-06-06

**Problem:** `Get.Parser.Shared/LRParserRunner.cs` uses `goto rerun` and `goto resolved` for error recovery control flow (lines 19, 87, 120). Makes the parse loop hard to follow.

**Resolution:**
1. Replaced `goto rerun` (reduce-then-rerun) with a `shouldRerun` flag in a `do-while` loop тАФ when a REDUCE action occurs, the flag is set and the loop re-enters action resolution without consuming the next token.
2. Replaced `goto resolved` (error recovery success) with a `recovered` flag тАФ when error recovery finds a valid action, control falls through naturally to action dispatch.
3. Replaced `continue` (skip problematic token after failed re-insertion) with `break` from the do-while тАФ same semantics, exits inner loop so the outer `foreach` advances to the next token.

All 13 LR parser and fuzz tests pass.

**Effort:** Medium (3-4 hours).

### 9. Unfinished EOF Handling in Lexer

**Status:** тЬЕ **Completed** тАФ 2026-06-06

**Problem:** `Get.Lexer/LexerBase.cs` line 168:
```csharp

```
When the lexer reaches EOF during tokenization, it simply stops instead of producing a proper EOF token.

**Resolution:**

### 12. Thread Safety: ThreadLocal<RegexParser>

**Status:** тЬЕ **Completed** тАФ 2026-06-06

**Problem:** `Get.RegexMachine.Shared/RegexParser.cs` line 14:
```csharp
static readonly ThreadLocal<RegexParser> Instance = ...
```
The `CreateEmptyNFAState` delegate property is mutable. If mutated after the thread-local is initialized, changes may not be visible to other threads (or worse, may race).

**Resolution:**
Made `CreateEmptyNFAState` a property with private setter. Added `Parse(string, Func<INFAState>)` overload that accepts the factory as a parameter, eliminating the mutation race window. Updated `RegexCompiler.NFA.cs` caller.

**Effort:** Small (30 min).
